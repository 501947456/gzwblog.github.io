2.2 环境变量说明

变量名 说明 默认值
POSTGRES_USER 数据库用户名 postgres

PGDATA 数据目录位置 /var/lib/postgresql/data/pgdata

2.3 数据持久化

  • 数据卷：映射 ./postgres-data 到容器数据目录（配置文件默认位于数据目录中）
  • 初始化脚本：.sql 或 .sh 脚本放入 ./init-scripts，首次启动自动执行

2.4 常见操作

操作 命令
连接数据库 docker exec -it postgres psql -U admin -d mydb
备份数据 docker exec postgres pg_dump -U admin mydb > backup.sql
恢复数据 cat backup.sql | docker exec -i postgres psql -U admin mydb
停止服务 docker-compose down
完全清除 docker-compose down -v
查看日志 docker-compose logs -f postgres
重启服务 docker-compose restart postgres

2.5 生产环境建议

  • 使用强密码并通过环境变量或密文管理
  • 配置定期备份策略（见 4.2 节）
  • 限制容器资源使用（memory, cpu）
  • 配置适当的健康检查
  • 开启 SSL 连接
  • 配置文件位于数据目录中，可通过数据卷持久化

三、命名规范

3.1 数据库对象命名

对象 规范 示例
数据库 snake_case order_management
表 snake_case，复数或业务前缀 users, sys_orders
列 snake_case user_name, created_at
索引 idx_表名_列名 idx_orders_user_id
约束 chk_表名_描述，uq_表名_列名 chk_users_age, uq_users_email
序列 seq_表名_列名 seq_users_id
视图 v_业务描述 或 vw_表名 v_active_users
函数 fn_功能描述 fn_calculate_total
触发器 trg_表名_动作 trg_orders_after_insert

3.2 SQL 编码规范

3.3 命名注意事项

  • 避免使用保留字作为对象名
  • 表名和列名使用单数形式，视图可使用复数
  • 布尔字段使用 is_ 或 has_ 前缀
  • 时间字段使用 _at 后缀
  • 外键字段使用 _id 后缀

七、配置参考

7.1 关键配置参数说明

连接配置

参数 默认值 说明 推荐值
max_connections 100 最大客户端连接数 200-500（根据内存调整）
superuser_reserved_connections 3 超级用户保留连接数 3
tcp_keepalives_idle 0 TCP保活空闲时间(秒) 60
tcp_keepalives_interval 0 TCP保活间隔(秒) 30
statement_timeout 0 语句超时时间(毫秒) 30000（生产环境建议设置）

内存配置

参数 默认值 说明 推荐值
shared_buffers 128MB 共享缓冲区大小 机器内存的25%
work_mem 4MB 排序/哈希操作内存 64-256MB
maintenance_work_mem 64MB 维护操作内存(VACUUM/ANALYZE) 256-512MB
effective_cache_size 4GB 操作系统缓存估计值 机器内存的75%
temp_buffers 8MB 临时缓冲区大小 64MB

写入配置（WAL）

参数 默认值 说明 推荐值
wal_level replica WAL日志级别 replica（流复制用）
fsync on 强制同步写入 on（安全，不建议关闭）
synchronous_commit on 同步提交模式 on
wal_buffers 4MB WAL缓冲区大小 16-64MB
checkpoint_timeout 5min 检查点超时时间 10-30min
checkpoint_completion_target 0.5 检查点完成目标 0.7-0.9
max_wal_size 1GB WAL最大 size 2-4GB
min_wal_size 80MB WAL最小 size 256MB

日志配置

参数 默认值 说明 推荐值
log_destination stderr 日志输出目标 csvlog（便于分析）
logging_collector off 是否启用日志收集器 on
log_directory log 日志目录 pg_log
log_filename postgresql-%Y-%m-%d 日志文件名格式 postgresql-%Y%m%d
log_rotation_age 1d 日志轮转时间 1d
log_rotation_size 10MB 日志轮转大小 100MB
log_min_duration_statement -1 记录慢查询阈值(毫秒) 1000（生产环境）
log_checkpoints off 记录检查点 on（调试用）
log_connections off 记录连接 off（生产环境）
log_disconnections off 记录断开连接 off（生产环境）
log_lock_waits off 记录锁等待 on
log_temp_files -1 记录临时文件大小 0（记录所有）

慢查询记录配置

参数 默认值 说明 推荐值
log_min_duration_statement -1 慢查询阈值(毫秒)，-1表示关闭 1000（生产环境）
log_statement none 记录哪些类型的语句 none（用慢查询日志即可）
log_duration off 记录每个语句的执行时间 off
log_line_prefix %m [%p] 日志行前缀格式 %m [%p] %q%u@%d

pg_stat_statements 扩展（推荐）

慢查询分析示例

查询规划配置

参数 默认值 说明 推荐值
random_page_cost 4.0 随机页面读取成本 1.1（SSD）
seq_page_cost 1.0 顺序页面读取成本 1.0
effective_io_concurrency 1 有效IO并发数 200（SSD）
default_statistics_target 100 统计信息目标值 100-500
constraint_exclusion partition 约束排除 partition

超时配置

参数 默认值 说明 推荐值
idle_session_timeout 0 空闲会话超时 3600000（1小时）
idle_in_transaction_session_timeout 0 空闲事务超时 300000（5分钟）
lock_timeout 0 锁等待超时(毫秒) 10000
deadlock_timeout 1s 死锁检测间隔 1s

7.4.3 常见配置修改场景

场景 修改方式 生效方式
调整慢查询阈值 ALTER SYSTEM SET log_min_duration_statement = 1000; 热加载
修改连接数 ALTER SYSTEM SET max_connections = 200; 重启
开启慢查询日志 修改 postgresql.conf 并挂载 重载或重启
调整内存参数 修改 postgresql.conf 中的 work_mem 热加载
