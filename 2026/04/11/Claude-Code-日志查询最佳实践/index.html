6. 最佳实践

6.1 查询优化建议

6.1.1 使用合适的时间范围

建议 说明
避免全量扫描 尽量指定明确的时间范围
利用索引优势 ES 按时间创建索引时，指定时间可跳过无效索引
合理设置 LIMIT 大量结果时使用分页或 LIMIT

反例：

正例：

6.1.2 善用 STATS 预聚合

反例： 先查询全部再客户端聚合

正例： 服务端聚合减少数据传输

6.1.3 合理使用字段

建议 说明
使用 keyword 类型精确匹配 service = "gateway" 而非 service LIKE "%gateway%"
避免对 text 字段全文搜索 使用 keyword 或配置专门搜索字段
利用 _source 过滤字段 减少网络传输

8.1 查询性能对比

查询方式 数据量 耗时 推荐场景
FROM logs-* LIMIT 10000 全索引扫描 慢 不推荐
WHERE time > NOW()-1h LIMIT 100 限定时间 快 日常查询
STATS BY hour 预聚合 快 统计分析
